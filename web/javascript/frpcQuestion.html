frpc相关错误解决方法 ​

frpc客户端启动 lookup 错误 ​

报错 ​

frpc 客户端启动 login to server failed: dial tcp: lookup 错误

shell
2022/02/25 18:20:40 [W] [service.go:105] login to server failed: dial tcp: lookup ddns.t3cc.cn on 218.104.111.122:53: read udp 192.168.33.77:56941->218.104.111.122:53: i/o timeout

解决 ​

将frpc.ini 中的server_addr的域名改成ip地址（ping命令或获取）即可，server_addr = 157.0.98.78

shell
server_addr = ddns.t3cc.cn // 例如我对外访问的网址实际的IP是：157.0.98.78
server_port = 7000

启动命令 ​

shell
./frpc -c ./frpc.ini

frpc客户端启动提示proxy name已使用 ​

报错 ​

shell
2022/02/25 18:26:33 [W] [control.go:179] [c41966a7f79085f3] [web02] start error: proxy name [web02] is already in use

解决 ​

就是代理名称proxy name已经被占用了，修改frpc.ini中的web02名称就行，改个其他的即可。

shell
[web02] // 这里修改，如改为 [web03]
type = http
local_ip = 127.0.0.1
local_port = 9528
subdomain = web02
